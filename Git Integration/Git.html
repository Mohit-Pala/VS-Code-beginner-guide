Git/Github

Install

Download Git for your desired operating system from https://git-scm.com/downloads. Run the installer for said program. the user will utilize the VSCode integrated terminal or the computer’s command prompt to type ‘git init’. This makes a new Git repository. To ensure that Git is being installed correctly, check the directory for the project being worked on and make sure the terminal has the same path before running ‘git init’.

Staging Changes

After making changes to your code, you can stage the changes to be included in the next commit. To do this, simply select the source control icon, then select the ‘+’ button next to the files you want to stage. You can also type "git add" without the quotation marks into the terminal.

Committing Changes

Once the changes have been staged, you can commit them to the repository. To commit your changes, at the top of the source control panel enter a commit message in the text box and select the check mark directly above that. You can also type "git commit -m "example"" into the terminal. Replace example with your message and remove the outer quotation marks.

Pushing and Pulling

If multiple people are working on the same code by using a remote repository, you’ll need to push your changes to said repository and pull changes from it as well. Pushing and pulling from the repository can be done by selecting the three dot - drop down menu at the top of the source control panel. From there, you’ll be able to see options such as pull, push, fetch, clone, and branch.

Branching and Merging

If you’ll be implementing new features or fixing bugs, Git allows you to work on multiple branches of the code simultaneously without affecting the main codebase. Branching and merging can both be done from the source control panel.
