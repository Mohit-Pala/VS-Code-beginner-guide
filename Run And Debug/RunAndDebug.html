Run and Debug

Run and Debug is self explanatory, as it runs code as well as debugs it. It offers four options to do so, “Step over”, “Step into”, “Step out” and “Continue” which can all be run through VS Code.

  1. Step over means to run a subroutine or branch without going through it step by step.
  2. Step into means to go into a subroutine or branch so you can go through it step by step.
  3. Step out means to run the rest of the subroutine or branch and jump out of it.
  4. Continue means to run everything automatically.

You can also “Run without debugging” which is once again self explanatory. The debugger will help you go through your code and find logical errors, but it does not always work. You can use the built in VS Code debugger or download and use another.
